## CONTEXT SNAPSHOT

### 0. PROJECT OVERVIEW

**What We're Building**

IndustryLabs is a B2B AI tool marketplace organized by **industry verticals and use cases**, starting with HR & Talent, then expanding to L&D and Customer Support.

Unlike generic AI tool directories that organize tools by broad categories, we organize by industry-specific workflows that buyers actually care about.

---

**Target Customers**

**Buyers**: HR/Talent/L&D/CS leaders in 50-500 employee companies (GTM focus)

- Need AI tools but lack time/expertise to evaluate dozens of options
- Care about compliance (GDPR, EEOC) and integrations with existing stack
- Want curated recommendations, not overwhelming lists

**Note on Company Size**: Platform directory supports all company sizes as filters (1-50, 51-200, 201-500, 500+ employees), but Year 1 go-to-market and content specifically target 50-500 employee mid-market companies.

**Vendors**: B2B SaaS/AI tool providers in People/Productivity space

- Want qualified, high-intent enterprise leads with full context (budget, timeline, company size)
- Tired of passive directory listings that generate no actionable leads
- Willing to pay for actual lead generation, not just visibility

**Geography (Year 1):** Focus on English-speaking markets (UK, US, Canada, Australia). No non-English localization or country-specific versions in Year 1.

---

**Core Product (Phase 1: HR Only)**

**Curated Tool Directory**

- HR AI tools organized by use case (Recruiting, Onboarding, Performance, Engagement, Analytics)
- Further filtered by company size, compliance requirements, and integrations
- Compliance information (GDPR, EEOC, SOC 2, HIPAA where relevant)
    - **Important**: All compliance information is vendor-supplied and represents indicative guidance only. We do not provide legal advice or formal certification.

**Request Board** (Core Differentiator)

- Buyers describe their specific situation, budget, and constraints
- We match them to 2-3 relevant vendors and provide introductions
- **Scope**: We provide high-intent introductions and facilitate initial contact. We do not guarantee conversions, negotiate deals, or act as a transaction intermediary.
- Vendors receive qualified leads with full context (not anonymous clicks)
- Creates network effects: More buyers â†’ more vendor value â†’ more vendors â†’ better matching â†’ more buyers

**AI-Assisted Content Engine**

- Tool comparisons ("Greenhouse vs Lever"), "Best AI for X" guides, buyer playbooks
- Generated from structured Airtable schema using Claude API
- Workflow: AI generates draft, then human review (20-30 mins), then publish
- Saves ~60% time vs manual writing (2.5 hrs/article vs 6-8 hrs fully manual)

---

**How It Scales (Phase 2-3: L&D and CS)**

**Multi-Vertical Expansion via Vendor Overlap**

- 70% of HR vendors also serve L&D (performance tools, engagement platforms)
- 65% of HR vendors also serve CS (engagement, analytics, workforce planning)
- Once HR validated, we reuse vendor relationships, content templates, and platform infrastructure
- Expansion cost per vertical: approximately 30% of initial HR investment

**Vendor Self-Serve Portal** (Month 9-12, only if triggers met)

- Vendors update their own profiles, pricing, features, and release notes
- Changes automatically flow into listings, comparisons, and guides
- Expected to reduce manual update workload significantly (target: over 50% reduction), with exact efficiency validated through real vendor usage patterns before Month 9
- **Implementation note** (high-level only): Built as a simple authenticated vendor area within the existing Next.js app (backed by Airtable), not a separate product

## AI Pair-Programming SOP

When working on any dev task, use AI as follows:

**1. CLARIFY (2-5 minutes)**

- Paste relevant code/files into assistant

**2. PLAN (Use GPT-5, 5-10 minutes)**

- Break task into 3-7 concrete steps
- Identify risks, edge cases, tests needed
- Confirm which files/components will be touched
- Output: Short checklist

**3. IMPLEMENT (Use Claude Code, 15-30 minutes)**
Open repo in VS Code with Claude Code:

- Navigate file tree and apply planned edits
- Keep changes scoped to agreed checklist
- Run `npm run lint` and `npm run build` locally
- Fix any issues before proceeding

**4. REVIEW (Use GPT-5, 5-10 minutes)**
Paste final diff or key files into GPT-5:

- Request code review (readability, safety, performance)
- Get suggestions for better naming, abstractions, tests
- Iterate if needed

**5. CROSS-CHECK (Optional, Use Alternative Model, 5 minutes)**
For security, performance-critical, or tricky algorithms:

- Get second opinion from Gemini or another model
- Focus on specific concerns

**Any AI assistant helping with IndustryLabs should:**

- Respect this SOP
- Clearly state whether they are in **PLAN**, **IMPLEMENT**, or **REVIEW** mode
- Never skip the review step for complex changes
